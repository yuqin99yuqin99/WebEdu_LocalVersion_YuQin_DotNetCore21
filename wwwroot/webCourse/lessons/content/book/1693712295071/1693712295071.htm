案例 5‑7 ASP.Net基于SK.Net使用Prompt语言互动AI

namespace SemanticKernel_AI_LLM_Try.Controllers

{

    public class AskGPTTranslationController : Controller

    {

        [HttpGet(Name = "AskGPTTranslation")]

        public async Task<string> AskGPTTranslation(String inputString)

        {

            IKernel iKernel = Kernel.Builder.Build();//新生一个IKernel对象类型的iKernel对象实例

            String promptForTranslation = "请将'"+inputString+"'翻译成英文。";//新生一个String对象类型的promptForTranslation对象实例,作为AI大语言模型的Prompt语言。在此的Prompt语言调用AI大语言模型进行中英翻译。当然也可进一步调用其他各种功能。

            ISKFunction llm = iKernel.CreateSemanticFunction(promptForTranslation); //创建一个 Function实现模型的调用。

            SKContext? responseFromLLM = await llm.InvokeAsync(inputString);

            return responseFromLLM.Result;

        }

    }

}

案例 5‑8互动AI的Prompt语言

目前主要使用Prompt语言互动AI。Prompt是当前计算机领域最接近人脑语言的语言[[6]]。

(1) Prompt语言的基本组成

·        指令（Instruction） 。即，希望AI模型完成的一个任务或者指令。

·        情景（Context） 。即，上下文信息，这些信息可以引导AI模型生成更到位的回答。

·        输入数据（Input Data）。 即，输入提供给AI模型的数据，AI模型处理该数据，生成所期望的回答。

·        输出提醒（Output Indicator） 。即， AI模型输出回答的提醒，例如，回答的类型或格式，等等。

(2) Prompt语言的注意事项

·        从简单开始：设计提示是一个迭代过程，需要大量的实验才能得到最佳结果。可以从简单的提示开始，随着目标越来越明确，逐渐添加更多关键词、上下文，从而获得AI模型的更到位回答。

·        指令很重要：可以使用指令为各种简单任务设计有效的提示，从而指示AI模型实现目标回答，例如“撰写”，“分类”，“总结”，“翻译”，“排序”等等。需要进行大量的实验，以了解什么效果最佳。尝试不同的指令、关键词、上下文、数据，比较AI模型回答的最佳表现。

·        具体性：提示越具体、越详细，AI模型的回到就越到位。

·        需要而不是不需要：设计提示时，避免说不要做什么，而是说需要做什么，从而获得AI模型的更到位的回答。

例如，通过ChatGPT了解什么是是Prompt Engineering，正常情况下可能会这样提问：

Prompt：解释一下prompt engineering的概念，简短一点，几句话就行，不要太过描述性。

AI：Prompt engineering是一种技术，用于设计和优化用于训练AI模型的提示(Prompt)，以提高模型的性能和效果。它需要考虑任务目标、设计清晰简洁的Prompt、优化Prompt效果和性能等因素。通过正确的Prompt设计，可以训练模型生成符合期望的输出，并提高模型的性能和准确性。

又例如，如果是中学生通过ChatGPT了解什么是Prompt Engineering，可能会这样提问：

Prompt：用2-3句话向中学生描述一下Prompt engineering（注：可以发现，当提示要求向中学生描述的时候，AI模型给出了专业术语更少的回答）

AI：Prompt engineering是一种通过设计指令来教A1做任务的方法。通过设计好的指令，AI能更好地理解任务，从而做出更好的回答。

(3) Prompt语言的标准化思考

不同的AI，例如ChatGPT、LLaMA、ChatGLM、绘画AI等等，Prompt也可能有所不同，所以，Prompt的标准化发展值得思考。
