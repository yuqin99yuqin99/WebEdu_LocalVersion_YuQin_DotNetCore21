1.4  代码词法：标识符；名称、ID

1.         标识符涵义

标识符是名称、ID的统称。标识符属于非关键词，是用作名称、ID的非关键词。代码中所定义的类型、对象，及其成员（属性、方法等），都是通过标识符进行指代的。例如，示例 1‑1中代码“class MyClass{;}”中的MyClass是所定义的类的标识符。

标识符用于指代类型、对象等，所以，同一名称空间中不能有重复的标识符，或者说，整个软件中不能存在重复的“名称空间.标识符”，而必须唯一。

思考：关键词能否用作标识符？

2.         标识符规范性

标识符要尽量体现真实意义，这样便于与他人交流，便于维护。同时，标识符可能需要由多个词复合而成，此时通常遵守如下几种规则中的一种，可结合示例代码自主体会。

l  Pascal规则：例如，MyClass这一标识符是遵守Pascal规则的，即，复合词的第一部分My首字母大写，第二部分Class首字母也大写，各部分都首字母大写。名称空间名称、类型名称、类型成员名称（属性名称、方法名称）一般选用Pascal规则。

l  Camel规则：例如，myClass这一标识符是遵守Camel规则的，即，复合词的第一部分my首字母小写，第二部分Class首字母大写，随后的各部分都首字母大写。类型实例化出的对象名称一般选用Camel规则。

1.5  代码词法：运算符、表达式

对象通过运算符进行运算（对象、运算符所形成的代码序列称为表达式）。例如示例 1‑1中代码 “Property1+Property2”中的“+”是进行加法运算的运算符，在此乃是对MyClass类型对象的Property1属性所返回的对象，与Property2属性所返回的对象，进行加法运算。 “Property1+Property2”代码块则称为一个表达式。

1.6  代码词法：注释

C#注释是指不真正用作C#代码的代码。这些代码一般用于C#代码的注解、说明，以便交流维护，所以称为注释。C#支持三种形式的注释：单行注释、多行注释、XML注释。

1.         单行注释

单行C#代码的注释,以字符“//”开头，并延续到本行的结尾。例如示例 1‑1中代码“//以上是一个.cs文件中C#代码的基本框架。”是一个单行注释。

2.         多行注释

多行C#代码的注释,以字符“/*”开头，并以字符“*/”结束，其间可以跨越多行。例如如下注释跨越两行：

/*以上是一个.cs文件

中C#代码的基本框架。*/

3.         XML注释/XML文档注释

C#代码的XML注释,又称为C#代码的XML文档注释，以字符“///”开头，还必须结合一些预定义的XML标记（如<summary>、<param>等），基于XML语法使用（基于立体化丛书的理念，XML请参见前期丛书，如参考文献….）。

/// <summary>这是对数据库初始化样本数据的方法。</summary>

/// <param name="musicStoreDbContext">以数据库上下文类型的对象作为参数/// </param>

其中的<summary>标记之间的内容将显示为摘要的内容，<param name="musicStoreDbContext">标记之间的内容将显示为各个参数的说明。并且将在出现在Inteligence部分，以便调用者理解使用（即时在调用二进制的，而非源代码的组件时也能查看到），就如调用.Net内置的大量.dll组件文件中的类型。

1.8  代码句法：语句、语句块

1.3 ~1.6 节叙述了C#词法。因为单词构成句（即语句），所以还必须理解C#句法。

1.         语句、语句块

语句由一系列单词构成，并以分号“;”结束。一个语句作为一个整体执行一次运算。例如示例 1‑1中语句“return Property1+Property2;”等。如果一个语句只包含分号“;”则表示是空语句，空语句可理解为不做任何事情。

语句块是由大括号“{}”括起来的一系列语句。语句块中的所有语句作为一个整体使用。例如示例 1‑1中如下代码是一个语句块，该语句块是MyClass类定义的整体代码。

internal class MyClass

    {

        internal Int32 Property1 { get; set; }

        internal Int32 Property2 { get; set; }

        internal Int32 Mothod1()

        {

            return Property1+Property2;

        }

    }

上述代码中所示，语句块可以嵌套，嵌套时“{}”必须互相匹配。

2.         语句类别

l  从关键词的视角分类

语句一定是由关键词来定义的，所以可从关键词的视角理解语局。如，if…else…分支语句、try…catch…finally异常处理语句、checked…unchecked溢出检查语句，等等，非常丰富，具体涉及时详述。

l  从功能的视角分类

语句一定是面向一定功能的，所以可从功能的视角理解语局。如，流程控制语局（if…else…分支语句、switch分支语句、for循环语句、foreach循环语句、while循环语句、do循环语句）、异常处理语句（try…catch…finally异常处理语句）、溢出检查语句（checked…unchecked溢出检查语句），等等。下述主要从此视角，概述流程控制语局（1.9 节），其他功能语句将在具体涉及时详述。

            }

}

l  代码“Int32 int32Number = new Int32();int32Number =7;”表示.Net内置的System.Int32类型实例化出对象int32Number，并赋值7。

l  代码“switch(int32Number)”表示将int32Number对象作为切换判断。.

l  代码“case 0: { return this.Content("星期天"); }”表示，如果上述int32Number对象的值等于0这种情形，那么将执行语句块“{return this.Content("星期天");}”。其他几个case语句是类似的，无需赘述。

l  代码“default: { return this.Content("没有这样的星期日子"); }”表示如果前面所有条件都不满足(即int32Number对象的值不是0、1、2、3、4、5、6这些情形)，那么就属于default这一默认情形，将执行语句块“{return this.Content("没有这样的星期日子");}”。

l  示例中int32Number赋值2（即case 2这一情形），所以执行语句块“{return this.Content("星期二");}”，因为this.Content是指自定义的TrySwitchController类型从.Net内置的System.Web.Mvc.Controller类型继承而来的内置方法，该方法此时返回"星期二"字符串，所以如图 1‑4所示。

l  可见，switch流程控制语句有可能实现任意想要的分支流程控制。

思考：switch分支语句与前述if…else…分支语句的异同。

l  代码“List<String> tryList = new List<String>();”表示.Net内置的System.Collections.Generic.List<T>泛型类实例化出对象tryList，tryList对象此时是一系列字符串对象的列表集合。

l  代码“Int32 int32Temp = 0;”表示.Net内置的System.Int32类型实例化出int32Temp对象，并赋值0。

l  代码“while (int32Temp<3)”表示当int32Temp小于3时（即int32Temp<3表达式为真，所以int32Temp<3表达式在此又常称为“循环条件”），执行随后的代码块“{stringForAllWeekDay = stringForAllWeekDay + tryList[int32Temp]+”;”;int32Temp = int32Temp + 1;}”，否则不执行。

l  代码块“{stringForAllWeekDay = stringForAllWeekDay +tryList[int32Temp]+”;”;int32Temp = int32Temp + 1;}”表示，上述stringForAllWeekDay字符串对象的值+tryList对象集合中的第int32Temp个字符串对象的值，然后int32Temp对象的值加1（所以int32Temp在此又常称为“计数器”，计数器每次增加的值又常称为“步幅”，在此的步幅为1）。因为int32Temp对象的值0、1、2递增，所获取的tryList对象集合中的字符串对象位置也是递增的，也即在循环。当int32Temp=3后，不再执行此代码块，也即跳出循环。

l  最后执行语句块“{return this.Content(stringForAllWeekDay);”。因为this.Content是指自定义的TryWhileLoopController类型从.Net内置的System.Web.Mvc.Controller类型继承而来的内置方法，该方法此时返回上述stringForAllWeekDay字符串对象的值，所以如图 1‑5所示。

l   可见，while流程控制语句有可能实现任意想要的循环流程控制。

思考：while循环语句与后述do…while…循环语句、for循环语句、foreach循环语句的异同。

5.      循环语句与递归语句

递归即调用自身。递归语句即一个语句块中的一个调用该语句块自身的语句。

用递归写一个关于求一个数的阶层的程序；越详细约好

用递归法计算n!用递归法计算n!可用下述公式表示：    n!=1         (n=0,1)    n×(n-1)!    (n>1)按公式可编程如下：long ff(int n){    long f;    if(n<0) printf("n<0,input error");    else if(n==0||n==1) f=1;    else f=ff(n-1)*n;    return(f);}main(){    int n;    long y;    printf("\ninput a inteager number:\n");    scanf("%d",&n);    y=ff(n);    printf("%d!=%ld",n,y);} 程序中给出的函数ff是一个递归函数。主函数调用ff 后即进入函数ff执行，如果n<0,n==0或n=1时都将结束函数的执行，否则就递归调用ff函数自身。由于每次递归调用的实参为n-1，即把n-1的值赋予形参n，最后当n-1的值为1时再作递归调用，形参n的值也为1，将使递归终止。然后可逐层退回。下面我们再举例说明该过程。设执行本程序时输入为5，即求5!。在主函数中的调用语句即为y=ff(5)，进入ff函数后，由于n=5,不等于0或1，故应执行f=ff(n-1)*n,即f=ff(5-1)*5。该语句对ff作递归调用即ff(4)。进行四次递归调用后，ff函数形参取得的值变为1，故不再继续递归调用而开始逐层返回主调函数。ff(1)的函数返回值为1，ff(2)的返回值为1*2=2，ff(3)的返回值为2*3=6，ff(4)的返回值为6*4=24，最后返回值ff(5)为24*5=120。
