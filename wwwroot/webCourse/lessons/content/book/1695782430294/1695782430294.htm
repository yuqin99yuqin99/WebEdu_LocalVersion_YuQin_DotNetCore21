VS对象浏览器中可查看到System.Object内置如下方法（没有内置属性）（“面向动态的方法/函数”可以变换成为“面向静态的属性/数据”）[[i]]。复合数据读写的对象类型的System.Object，是由简单数据读写的对象类型封装而成的属性/方法，所以，属性/方法支持各种数据计算，即，5.7.3节所述的定性计算、5.7.3节所述的布尔计算、5.7.5节所述的定量计算。

l  public Object()：System.Object对象实例被生成时时将调用的构造方法。

l  ~Object()：System.Object对象实例被销毁时将调用的析构方法。

l  public static System.Boolean Equals(object objA, object objB)：比较objA、objB两个对象是否拷贝相等。参数:objA:要比较的System.Object类型的对象之一。objB:要比较的System.Object类型的对象之二。返回值:如果两个对象相等，则返回true（简单数据读写的对象类型System.Boolea的值）；否则返回fals（简单数据读写的对象类型System.Boolea的值）。此方法与objA==objB表达式是等同的。

l  public virtual System.Boolean Equals(object obj)：判断作为参数传入的System.Object类型的对象是否等于当前的System.Object类型的对象。参数:obj:作为参数传入的System.Object类型的对象。返回值: 如果两个对象相等，则返回true（简单数据读写的对象类型System.Boolea的值）；否则返回false（简单数据读写的对象类型System.Boolea的值）。

l  public static System.Boolean ReferenceEquals(object objA, object objB)：比较objA、objB两个对象是否引用相等。参数:objA:要比较的System.Object类型的对象之一。objB: 要比较的System.Object类型的对象之二。返回值:如果objA与objB是相同的实例，或者如果二者都为null，则返回true（简单数据读写的对象类型System.Boolea的值）；否则返回false（简单数据读写的对象类型System.Boolea的值）。此方法与objA==objB表达式是等同的。

l  public virtual System.Int32 GetHashCode()：返回值:当前对象的哈希代码（简单数据读写的对象类型System.Int32的值）。

l  public System.Type GetType()：返回值:当前对象的类型（复合数据读写的对象类型System.Type的单态/多态的实例）。

l  protected System.Object MemberwiseClone()：返回值:当前对象的浅拷贝[[ii]]（继承拓展的树根对象类型System.Object的单台/多态的实例）。

l  public virtual System.String ToString()：返回值:当前对象所转化而来的字符串（简单数据读写的对象类型System.String的值）。

 

 


[[i]]https://learn.microsoft.com/en-us/dotnet/api/system.object?view=net-7.0

[[ii]]对象的复制，分为浅度复制/深度复制。一个对象的成员有可能是拷贝对象类型，有可能是引用对象类型。对象浅度复制的时候，对于拷贝对象类型成员，会复制其本身(拷贝对象类型变量本身包含了所有数据，复制时进行按位拷贝)；对于引用对象类型成员(注意它会引用另一个对象)，仅仅复制引用，而不创建其引用的对象。结果就是：新对象的引用成员和复制对象的引用成员指向了同一个对象。深度复制就是将引用成员指向的对象也进行复制。实际的过程是创建新的引用成员指向的对象，然后复制对象包含的数据。深度复制可能会变得非常复杂，因为引用成员指向的对象可能包含另一个引用对象类型成员，等等。浅度复制调用类型内置的MemberwiseClone()就可以实现。深度复制需要自创方编写代码实现，可有多种途径，建议使用序列化方法最为简易。序列化是指任何类型的对象状态，都可序列化为二进制或XML格式文件存储起来（以便在需要时反序列化而还原对象状态，实现还原使用），或，序列化为二进制或XML格式的数据流而传输给其他应用程序（以便其他应用程序反序列化而还原对象状态，实现数据交换）。例如，通过将对象序列化为文件，可在不同的应用程序打开。又如，通过将对象序列化到剪贴板，可在不同的应用程序之间共享对象，等等。.Net的System.Runtime.Serialization名称空间包含可用于序列化和反序列化的对象类型，System.Xml.Serialization名称空间包含可用于将对象序列化为XML格式的文件或流的对象类型；System.IO名称空间包含读写文件和数据流的对象类型，以及操作目录/文件的对象类型，以便调用。
