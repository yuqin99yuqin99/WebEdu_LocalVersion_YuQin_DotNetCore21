实际使用时，常将此类型的数据保存为数据库之外的一个文件，数据库中以nchar[(n)]数据类型的方式存储该文件的路径的字符串，从而可定位到该文件，读写文件中的数据。

时间日期数据类型：SQL Server提供了time（格式如12:35:29. 1234567）、date（格式如2007-05-08）、smalldatetime（格式如2007-05-08 12:35:00）、datetime（格式如2007-05-08 12:35:29.123）、datetime2（格式如2007-05-08 12:35:29. 1234567）、datetimeoffset（格式如2007-05-08 12:35:29.1234567 +12:15），与.Net内置的System.DateTime类型、System.DateTimeOffset类型具有一定的对应性，但鉴于日期时间格式繁杂，转化为数值或字符串形式也可表示。

uniqueidentifier数据类型：16字节GUID全球唯一标识符的数据类型（.Net中尚未内置类似的类型）。手工录入GUID较麻烦，可用SQL的newid()函数生成。主键列如果使用了该数据类型，当该表复制了多个副本后，这些副本中的该列不是相同，而是自动变化为全球唯一，而其他列则是相同的。如果全局唯一性并不是必须的，而只需要一个表中唯一性地连续递增的键，则可以考虑使用列的IDENTITY属性。数字类型有个identy(“标识”)属性，设置好“标识种子”、“标识递增量”，每增加一条记录时，这个字段的值就会在最近的一个标识值基础上自动增加。

timestamp数据类型：自动生成的整个数据库中唯一的二进制数的数据类型（.Net中尚未内置类似的类型），注意timestamp值与时间并没有任何关系。一个表只能有一个timestamp数据类型的列，含有timestamp数据类型的列的表，每次修改时，timestamp数据类型的列的值会自动更新。在多用户并发访问数据库时，通过比较两次读出的timestamp数据类型的列的值，就可知道该行数据是否已被其他用户修改过。

最本质的数据类型（定性、定量、布尔）→面向统计领域数据类型（定类、定序、定距、定比）（这四种类型是从低到高的递进关系，高级的类型可以用低级类型的分析方法来分析，但是反过来却不适合）。

A.   定性数据类型→定类数据类型（Norminal Data Type）

可以分类（ = 和 ≠ ），但不能排序。定类就是给数据定义一个类别。这种数据类型能将所研究的对象区分开。例如把性别分成男、女。

B.   定性数据类型→定序数据类型（Ordinal Data Type）

可以分类（ = 和 ≠ ），可以排序（> 和 <），但不能（+ 和 - ）。定序变量的值不仅能够代表事物的分类，还能代表事物按某种特性的排序，但各个定序变量的值之间没有确切的间隔距离，只能排列出它们的顺序，而不能反映出大于或小于的数量或距离。例如，成绩等级可以分为优秀、良好、及格、不及格。

C.   定量数据类型→定距数据类型（Interval Data Type）

可以分类（ = 和 ≠ ），可以排序（> 和 <），可以（+ 和 - ），但不能（× 和 ÷ ）。定距变量没有绝对零点。定距变量的值之间可以比较大小，两个值的差有实际意义。但乘法是没有意义的。但是加减法有效。例如华氏温度：10，20，30，华氏度40不是20的两倍热。

D.   定量数据类型→定比数据类型（Ratio Data Type）

可以分类（ = 和 ≠ ），可以排序（> 和 <），可以（+ 和 - ），可以（× 和 ÷ ）。定比变量有绝对零点。定比变量除了具有定距变量的特性外，还具有一个真正的零点，因而它具有乘与除（×、÷）的数学特质。如A的体重是60kg，而B的体重是30kg，可以算出前者是后者的两倍重，因为其零点是绝对的。定比变量是最高测量层次的变量。
