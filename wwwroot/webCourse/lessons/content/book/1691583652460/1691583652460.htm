System.Numerics.Complex

 

 

System.Numerics.Complex 结构表示复数，即带实数部分和虚数部分的数字。 此类型支持一套标准的算术、比较、相等、显式和隐式转换运算符，以及数学、代数和三角方法（默认值的字面是0）。

https://learn.microsoft.com/zh-cn/dotnet/api/system.numerics.complex?view=net-7.0

System.Numerics 命名空间包含一组启用了 .NET SIMD 的类型。 SIMD (Single Instruction Multiple Data) 操作可以在硬件级别并行化。 这可以增加向量化计算的吞吐量，这在数学、科学和图形应用中很常见。在执行自定义 SIMD 算法之前，可以使用 Vector.IsHardwareAccelerated 来检查主机是否支持 SIMD（返回一个 Boolean）。 这并不能保证为特定类型启用 SIMD 加速，但表明某些类型支持它。SIMD 更有可能消除一个瓶颈但又暴露下一个瓶颈，例如内存吞吐量。 一般来说，使用 SIMD 的性能优势取决于具体的场景，在某些情况下，它的性能甚至可能比更简单的非 SIMD 等效代码更差。

启用了 .NET SIMD 的类型如下：

Vector2、Vector3 和 Vector4 类型，用于表示具有 2、3 和 4 Single 值的向量。

两个矩阵类型：Matrix3x2（表示 3x2 矩阵）和 Matrix4x4（表示 4x4 矩阵）。

Plane 类型，表示三维空间中的一个平面。

Quaternion 类型，表示一个用于对三维物理旋转进行编码的向量。

Vector<T> 类型，表示指定数字类型的向量，并提供受益于 SIMD 支持的一组广泛的运算符。 Vector<T> 实例的计数是固定的，但其值 Vector<T>.Count 取决于执行代码的计算机的 CPU。

(2) “0/1的二进制”转“十进制”（不会出现误差）

把人脑多媒体语言数按权展开，相加即得人脑多媒体语言数。具体案例如下：

人脑多媒体语言数(10010.111)2 =1 * 24 + 0 * 23 + 0 * 22 + 1 * 21 + 0 * 20 + 1 * 2-1 + 1 * 2-2 + 1 * 2-3 =16 + 2 + 1/2 + 1/4 + 1/8 =(18.875)10

(3)  “十进制”转“0/1的二进制”（可能出现误差）

l   整数部分方法为（不会出现误差）：“十进制数值”除2取余法，即，“十进制数值”除2，余数为权位上的数，得到的商值继续除2，依此步骤继续向下运算直到商为0为止，最后一个余数到最后一个余数的序列就是转化而成的“0/1的二进制数值”。如下“十进制数值(18)10”转换为“0/1的二进制数值(10010)2”：

18/2=9…………0;

9/2=4…………1;

4/2=2…………0;

2/2=1…………0;

1/2=0…………1;

l  小数部分方法为（小数部分尾数不是5的可能出现误差）：将该数字乘以2，取出整数部分作为“十进制数值”表示的第1位；然后再将小数部分乘以2，将得到的整数部分作为“十进制数值”表示的第2位；以此类推，直到小数部分为0。 特殊情况： 小数部分出现循环，无法停止，则用有限位数的“十进制数值”无法准确表示一个小数，这也是在编程语言中表示小数会出现误差的原因.只能增加“十进制数值”长度提高表示的精度。 　下面我们具体计算一下0.6的小数表示过程。我们可以发现在该计算中已经出现了循环，如下(0.6)10转换为(1001100110011001……)2

0.6 * 2 = 1.2 ——————- 1

0.2 * 2 = 0.4 ——————- 0

0.4 * 2 = 0.8 ——————- 0

0.8 * 2 = 1.6 ——————- 1

0.6 * 2 = 1.2 ——————- 1 …………

(4)  “十六进制”互转“0/1的二进制”（不会出现误差）

例如：(0000)2↔(0)16；(0001)2↔(1)16；(0010)2↔(2)16；(0011)2↔(3)16；(0100)2↔(4)16；(0101)2↔(5)16；(0110)2↔(6)16；(0111)2↔(7)16；(1000)2↔(8)16；(1001)2↔(9)16；(1010)2↔(A)16；(1011)2↔(B)16；(1100)2↔(C)16；(1101)2↔(D)16；(1110)2↔(E)16；(1111)2↔(F)16；

(5)  “十六进制”互转“十进制”（可能出现误差）

可以根据上述，“十六进制”首先互转“0/1的二进制”然后互转“十进制”。
